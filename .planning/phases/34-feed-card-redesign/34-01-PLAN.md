<task type="checkpoint:decision" gate="blocking">
  <decision>Select Polaroid frame contrast style for dark theme</decision>
  <context>
The app uses a dark theme (#0F0F0F background). Light Polaroid frames will pop against this dark background. We need to decide how much contrast is desired.

The photo itself stays untouched - only the frame gets styled. The choice affects how "loud" the cards are in the feed.
</context>
<options>
<option id="cream">
<name>Full Contrast (Cream)</name>
<pros>Classic Polaroid look, frames really pop, nostalgic and authentic, makes photos feel special</pros>
<cons>High contrast could feel jarring, frames may draw attention away from photos</cons>
</option>
<option id="warmGray">
<name>Softened (Warm Gray)</name>
<pros>Balanced contrast, still reads as Polaroid, easier on eyes in dark mode</pros>
<cons>Less dramatic, may lose some of the iconic Polaroid punch</cons>
</option>
<option id="muted">
<name>Muted (Dark Off-White)</name>
<pros>Subtle Polaroid effect, photos remain star of show, cohesive with dark theme</pros>
<cons>May be too subtle, could lose Polaroid recognition</cons>
</option>
</options>
<resume-signal>Select: cream, warmGray, or muted</resume-signal>

<task type="auto">
  <name>Task 3: Implement Polaroid frame styling</name>
  <files>src/styles/FeedPhotoCard.styles.js, src/components/FeedPhotoCard.js</files>
  <action>
Based on the selected contrast option, update FeedPhotoCard to Polaroid style:

**FeedPhotoCard.styles.js changes:**

1. **Card container (Polaroid frame):**
   - Change `card` backgroundColor to selected polaroid color from decision
   - Set border radius to 4px (Polaroids have slight rounded corners)
   - Add subtle drop shadow for "laying on table" effect
   - Remove current shadowOpacity/shadowRadius, replace with:
     ```javascript
     shadowColor: '#000',
     shadowOffset: { width: 0, height: 4 },
     shadowOpacity: 0.25,
     shadowRadius: 8,
     elevation: 6,
     ```

2. **Frame padding (Polaroid proportions):**
   - Create wrapper inside card with padding: top 10px, left/right 10px, bottom 50px (thick bottom)
   - Photo sits inside this wrapper

3. **Photo container:**
   - Keep aspectRatio: 1 (square photo)
   - Remove any border radius (photos in Polaroids are crisp-edged)
   - Add very subtle inner shadow (inset effect) with overlay

4. **Bottom info section (replaces profile section + reaction bar):**
   - Position inside the thick 50px bottom padding
   - Layout: user info left, timestamp right
   - Font styling: Use a slightly handwritten feel (fontStyle: 'normal', but thinner weight)
   - Text color: polaroid.text for name, polaroid.textSecondary for timestamp
   - No profile photo - just text (Polaroid authenticity)

5. **Reactions:**
   - Position below user info in bottom section, or omit if space tight
   - Smaller, more subtle styling to not dominate the "handwriting" area
   - Use polaroid.textSecondary color

**FeedPhotoCard.js changes:**

1. Remove the profileSection (profile photo, user info block) from the top
2. Remove the separate reactionBar from bottom
3. Create a single bottom info area inside the Polaroid frame:

   ```jsx
   <View style={styles.polaroidBottom}>
     <View style={styles.userInfo}>
       <Text style={styles.displayName}>{displayName}</Text>
       <Text style={styles.timestamp}>{getTimeAgo(capturedAt)}</Text>
     </View>
     {reactionCount > 0 && (
       <View style={styles.reactions}>
         {topReactions.map(...)}
       </View>
     )}
   </View>
   ```

4. Keep the getTopReactions logic unchanged
5. Keep TouchableOpacity and onPress behavior unchanged

**Important - do NOT:**

- Add filters or effects to the photo itself
- Change the tap behavior
- Add username (just displayName for cleaner look)
- Make the bottom section too busy
