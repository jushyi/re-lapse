<task type="auto">
  <name>Task 1: Restructure colors.js with correct values</name>
  <files>src/constants/colors.js</files>
  <action>
Update colors.js with the following changes:

1. Change `background.primary` from '#0F0F0F' to '#000000' (pure black)
2. Change `background.secondary` from '#1A1A1A' to '#111111' (very subtle lift - the "barely visible card" effect)
3. Keep `background.tertiary` as '#2A2A2A' for nested elements that need more contrast
4. Add new `background.card` alias pointing to '#111111' for clarity

5. Add `icon` section with explicit icon colors:

   ```
   icon: {
     primary: '#FFFFFF',    // Default icon color
     secondary: '#888888',  // Muted icons
     tertiary: '#666666',   // Very muted icons
     inactive: '#555555',   // Inactive/disabled icons
   }
   ```

6. Ensure `brand.purple` is '#8B5CF6' (the primary interactive/highlight color)

7. Add `interactive` section for button/control states:

   ```
   interactive: {
     primary: '#8B5CF6',      // Primary buttons, active tabs, focused inputs
     primaryPressed: '#7C3AED', // Pressed state (darker purple)
     secondary: '#333333',    // Secondary buttons
     secondaryPressed: '#444444',
   }
   ```

8. Keep existing status colors but ensure consistency:
   - status.danger should be '#FF3B30' (iOS red)
   - status.ready should be '#22C55E' (green)

9. Update `border.subtle` from '#333333' to '#222222' (subtler on pure black)

Keep all other existing values for backwards compatibility. Add comments explaining the color system.

1. Add import at top: `import { colors } from '../constants/colors';`

2. Replace all hardcoded color values with constants:
   - '#000000' → colors.background.primary
   - '#FF3B30' → colors.status.danger (or colors.system.iosRed)
   - '#FFFFFF' → colors.text.primary or colors.icon.primary (depending on context)

3. Specifically update:
   - All `contentStyle: { backgroundColor: '#000000' }` → `contentStyle: { backgroundColor: colors.background.primary }`
   - All `sceneContainerStyle` backgrounds
   - Tab bar styling (background, activeTintColor, inactiveTintColor)
   - Header styling (backgroundColor, tintColor)
   - Notification badge color

4. Ensure the navigation theme uses colors:
   ```
   const navTheme = {
     dark: true,
     colors: {
       primary: colors.brand.purple,
       background: colors.background.primary,
       card: colors.background.secondary,
       text: colors.text.primary,
       border: colors.border.subtle,
       notification: colors.status.danger,
     },
   };
   ```

Do NOT change any navigation logic or screen configurations - only replace hardcoded colors with constants.
</action>
