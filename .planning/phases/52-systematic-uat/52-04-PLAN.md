<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Profile viewing and editing</what-built>
  <how-to-verify>
    **Step 1: View Own Profile**
    1. Go to Profile tab
    2. Verify profile photo, username, display name, bio display correctly
    3. Verify "Edit Profile" button appears
    4. Verify photo grid shows (empty if no photos yet)
    5. Verify album section shows (empty or monthly albums if exist)
    6. Verify Selects banner appears at top

    **Expected:**
    - All profile fields render correctly
    - No missing data or placeholder text
    - Layout is clean and aligned

    **Step 2: Edit Profile - Valid Changes**
    1. Tap "Edit Profile"
    2. Change display name to something new
    3. Update bio (add/edit text)
    4. Tap "Choose Photo" → select new photo from library
    5. Tap "Save"
    6. Verify returns to profile screen
    7. Verify changes are visible immediately

    **Expected:**
    - All fields editable
    - Photo picker works
    - Save button shows loading state
    - Changes persist after save
    - No errors during save

    **Step 3: Edit Profile - Edge Cases**
    1. Tap "Edit Profile" again
    2. Try clearing display name (leave empty)
    3. Tap "Save"
    4. Verify error message appears (display name required)
    5. Try entering very long bio (500+ characters)
    6. Verify character limit enforcement
    7. Try changing username to one with special characters (e.g., "user@123")
    8. Verify validation error (alphanumeric + underscore only)
    9. Tap "Cancel" to exit without saving

    **Expected:**
    - Empty display name shows error, doesn't save
    - Bio has character limit (from Phase 14 work)
    - Username validation works
    - Cancel discards unsaved changes

  </how-to-verify>

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Settings screens and functionality</what-built>
  <how-to-verify>
    **Step 1: Settings Navigation**
    1. Go to Profile → Settings (gear icon)
    2. Verify settings sections appear:
       - Account (Edit Profile, Delete Account)
       - Notifications (push notification toggles)
       - Privacy (Blocked Users)
       - Legal (Privacy Policy, Terms of Service)
       - Support (Help, Contact, About)
    3. Tap into each section, verify screen loads
    4. Tap back to return to settings

    **Expected:**
    - All sections accessible
    - No crashes or blank screens
    - Back navigation works from all screens

    **Step 2: Notification Settings**
    1. Go to Settings → Notifications
    2. Verify notification toggles appear (e.g., Friend Requests, Comments, Tags, Darkroom Ready)
    3. Toggle each setting on/off
    4. Verify toggle state updates immediately
    5. Exit and re-enter to verify state persisted

    **Expected:**
    - Toggles update immediately
    - State persists across navigation
    - No crashes when toggling

    **Step 3: Privacy - Blocked Users**
    1. Go to Settings → Privacy → Blocked Users
    2. Verify blocked users list appears
    3. If empty, verify empty state message
    4. If users blocked (from earlier testing), verify list shows them
    5. Try unblocking a user (if any blocked)
    6. Verify confirmation prompt appears
    7. Tap Cancel to keep user blocked

    **Expected:**
    - List loads correctly
    - Empty state shows if no blocked users
    - Unblock requires confirmation

    **Step 4: Legal - Privacy Policy & Terms**
    1. Go to Settings → Legal → Privacy Policy
    2. Verify policy text appears (updated for Flick from Phase 51)
    3. Scroll through, verify formatted correctly
    4. Tap back
    5. Go to Legal → Terms of Service
    6. Verify terms text appears (updated for Flick)
    7. Scroll through, verify formatted correctly

    **Expected:**
    - Both documents load fully
    - Text is readable, formatted correctly
    - No "Rewind" references (all changed to "Flick")

    **Step 5: Support - Help Screen**
    1. Go to Settings → Support → Help
    2. Verify help topics appear (FAQ-style)
    3. Tap a topic to expand
    4. Verify answer text appears
    5. Tap another topic
    6. Verify multiple topics can be expanded simultaneously

    **Expected:**
    - Help topics load
    - Expandable/collapsible works smoothly
    - Content is helpful and accurate

    **Step 6: Theme Palette Selection**
    1. Go to Settings → Account → Theme (or wherever palette selection is)
    2. Verify current theme palette is highlighted
    3. Tap a different palette option
    4. Verify UI colors update immediately (profile screen, buttons, etc.)
    5. Exit settings and verify theme persists

    **Expected:**
    - Palette selection UI shows all options
    - Selection updates colors app-wide immediately
    - Theme persists across app restarts
